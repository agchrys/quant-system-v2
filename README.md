# Quant-System v2 — 个人量化交易系统

> **五层架构** · 数据驱动 · 严格风控 · 自动迭代优化
>
> 从数据采集到策略执行，系统性拆解量化交易的每一个环节。

| 指标 | 目标 |
|------|------|
| 胜率 (Win Rate) | ≥ 60% |
| 年化收益率 (Annual Return) | ≥ 20% |
| 最大回撤 (Max Drawdown) | < 12% |
| 夏普比率 (Sharpe Ratio) | > 1.5 |

## 🚀 快速开始

### 1. 安装依赖

```bash
cd quant-system-v2
pip install -r requirements.txt
```

### 2. 运行策略

```bash
# 查看所有可用策略
python main.py --mode list-strategies

# 运行 v8.15 策略（年化 20.66%）
python main.py --strategy v8_15

# 一键运行完整流程：数据采集 → 因子计算 → 模型训练 → 回测 → 报告
python main.py --mode full
```

### 3. 自动迭代优化

系统自动搜索最优参数，多轮迭代直至达到目标：

```bash
python main.py --mode iterate

# 或使用脚本
bash scripts/run_pipeline.sh
bash scripts/auto_iterate.sh
```

### 4. 查看报告

在 `reports/` 目录下查看生成的 CSV 报告和 PNG 图表。

## 💡 各层详解

### Layer 1 — 数据基础设施

数据源使用 **AKShare**（免费开源 A 股数据接口）：

| 数据类型 | 来源 | 频率 | 用途 |
|----------|------|------|------|
| 日线行情 | AKShare | 日 | 量价因子计算 |
| 财务数据 | 财报（季报/年报） | 季度 | 基本面因子 |
| 资金流数据 | AKShare | 日 | 资金因子 |
| 指数数据 | 沪深300 / 中证500 | 日 | 基准对比、择时 |

数据清洗包括：复权处理、缺失值填充、停牌处理、时间对齐。首次运行自动缓存数据，后续增量更新。

### Layer 2 — 因子工程

**传统因子（20+ 核心因子）：**

| 类别 | 因子 | 说明 |
|------|------|------|
| 动量 | momentum_1m/3m/6m | 过去 N 日收益率 |
| 反转 | reversal_5d | 短期反转效应 |
| 均线 | ma_deviation | 收盘价偏离均线程度 |
| 波动率 | volatility_20d/60d | 历史波动率 |
| 流动性 | turnover_rate/change | 换手率及其变化 |
| 量比 | volume_ratio | 近期成交量相对水平 |
| 估值 | pe_rank, pb_rank | PE/PB 历史分位数 |
| 质量 | roe_factor | 净资产收益率 |
| 成长 | revenue_growth, profit_growth | 营收/利润增速 |
| 非流动性 | amihud | Amihud 指标 |

**因子处理流水线：**
1. 去极值（MAD 或分位数法）
2. 标准化（Z-score 或 Rank 排序）
3. 行业/市值中性化
4. 缺失值填充

### Layer 3 — 信号生成

**模型：LightGBM Lambdarank**
- 排序学习任务，直接优化 NDCG 指标
- 支持早停机制，防止过拟合
- 自动特征重要性排序

**市场择时模块：**

| 市场状态 | 仓位建议 | 选股偏好 |
|----------|----------|----------|
| 趋势上涨 | 高仓位（80-100%） | 高 Beta、动量强 |
| 震荡 | 中等仓位（40-60%） | 低波动、高股息 |
| 趋势下跌 | 低仓位（10-30%） | 防御性为主 |
| 高波动 | 极低仓位（0-20%） | 现金为王 |

### Layer 4 — 组合管理

**仓位分配：**
- 风险平价 (Risk Parity) 或均值-方差优化
- 单股权重 ≤ 10%，单行业 ≤ 30%
- 定期再平衡（按周/月）

**四层风控机制：**

```
第一层：个股止损         亏损 3% 止损，N 日不涨减仓
第二层：组合回撤控制     回撤 5%/7%/10% 依次降仓
第三层：波动率目标       超标自动降杠杆
第四层：异常检测         (预留) 新闻舆情检测
```

### Layer 5 — 回测与评估

**回测引擎特性：**
- 事件驱动，按日推进
- 支持滑点、佣金、印花税模拟
- A 股交易规则（T+1、整百股交易）

**性能指标：**
- 年化收益率、胜率
- 最大回撤（含峰值/谷值/恢复日期）
- 夏普比率、卡玛比率、索提诺比率
- 盈亏比、月度收益矩阵、滚动指标分析

**⚠️ 关键注意事项：**
| 项目 | 说明 |
|------|------|
| 手续费 | 万 1.5 佣金 + 千 1 印花税（A 股） |
| 滑点 | 按成交量的 0.1%-0.5% 估算 |
| 未来函数 | 严格确保 T 日只使用 T-1 及之前数据 |
| 生存者偏差 | 回测必须包含已退市股票 |

## 🔄 自动迭代机制

```
初始化 → 数据采集 → 因子计算 → 模型训练
  ↓                              ↓
  参数搜索 ← 回测评估 ← 组合优化 ← 信号生成
  ↓
  达到目标？→ 是 → 保存版本 + 生成报告
  ↓ 否
  继续下一轮迭代
```

**迭代参数：**
- 网格搜索或随机搜索
- 参数空间涵盖学习率、树深度、采样率、正则化等 8 个维度
- 早停机制：连续 3 次无改善自动终止

```
选股规则：
  1. PE 处于历史 20%-80% 分位数

  沪深 300 在 20 日均线上方 → 做多

风控：
  单股止损 5%，总回撤 10% 清仓
```

## ❓ 常见问题

**Q: 需要 AKShare token 吗？**
A: 不需要，AKShare 是免费开源接口。

**Q: 回测时间多长？**
A: 首次运行约 10-30 分钟（取决于股票数量和历史长度），后续使用缓存会快很多。

**Q: 支持实盘吗？**
A: 本系统主要用于研究和回测。如需实盘，可对接 QMT/Ptrade 等券商接口。

**Q: 能跑在 Windows 上吗？**
A: 可以，但建议使用 Linux/Mac 获得更好的性能。
